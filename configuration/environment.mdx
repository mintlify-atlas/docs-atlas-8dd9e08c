BioAgents is configured entirely through environment variables. This guide covers all available configuration options.

## Core Configuration

### Authentication & Security

<ParamField path="BIOAGENTS_SECRET" type="string" required>
  Shared secret for API authentication (JWT signing/verification and API key).
  
  Generate with:
  ```bash
  openssl rand -hex 32
  ```
  
  <Warning>Never commit this value to version control. Use a strong random value in production.</Warning>
</ParamField>

<ParamField path="AUTH_MODE" type="string" default="none">
  Authentication mode for the API.
  
  **Options:**
  - `none` - No authentication required (development only)
  - `jwt` - Require JWT tokens signed with `BIOAGENTS_SECRET` (production)
  
  <Info>x402/b402 payment authentication is controlled separately via `X402_ENABLED`/`B402_ENABLED` flags.</Info>
</ParamField>

<ParamField path="ALLOWED_ORIGINS" type="string">
  Comma-separated list of allowed origins for CORS requests.
  
  **Required in production** to prevent CSRF attacks. Leave empty for development (defaults to `localhost:3000`, `localhost:5173`).
  
  ```bash
  ALLOWED_ORIGINS=https://bioagent-platform.bioagents.dev,https://app.bioagents.xyz
  ```
</ParamField>

<ParamField path="MAX_JWT_EXPIRATION" type="number" default="3600">
  Maximum JWT expiration time in seconds. Tokens with longer expiration will be rejected.
  
  Default: 3600 (1 hour)
</ParamField>

### Server Configuration

<ParamField path="NODE_ENV" type="string" default="development">
  Node environment mode.
  
  **Options:**
  - `development` - Development mode with verbose logging
  - `production` - Production mode with optimized logging
</ParamField>

<ParamField path="PORT" type="number" default="3000">
  HTTP server port.
</ParamField>

### Database (Supabase)

<ParamField path="SUPABASE_URL" type="string" required>
  Your Supabase project URL.
  
  Find in: Supabase Dashboard > Settings > API > Project URL
</ParamField>

<ParamField path="SUPABASE_ANON_KEY" type="string" required>
  Supabase anonymous key for client-side operations.
  
  Find in: Supabase Dashboard > Settings > API > `anon` public key
</ParamField>

<ParamField path="SUPABASE_SERVICE_KEY" type="string">
  Service role key for backend operations (bypasses Row Level Security).
  
  **Required for production** when RLS is enabled.
  
  Find in: Supabase Dashboard > Settings > API > `service_role` key
  
  <Warning>Keep this secret. It has full database access.</Warning>
</ParamField>

<ParamField path="SUPABASE_FULL_URL" type="string">
  Full database connection URL for migrations.
  
  Used by migration containers. Format: `postgresql://user:password@host:port/database`
</ParamField>

<ParamField path="CHUNK_OVERLAP" type="number" default="200">

<ParamField path="VECTOR_SEARCH_LIMIT" type="number" default="20">

</ParamField>

<ParamField path="RERANK_FINAL_LIMIT" type="number" default="5">

## Job Queue (BullMQ)

<ParamField path="USE_JOB_QUEUE" type="boolean" default="false">
  Enable job queue with BullMQ and Redis.
  
  - `false` - In-process execution (simple, for development)
  - `true` - Queue jobs in Redis (for production, supports horizontal scaling)
</ParamField>

<ParamField path="REDIS_URL" type="string" default="redis://localhost:6379">
  Redis connection URL for BullMQ.
  
  Required when `USE_JOB_QUEUE=true`.
</ParamField>

<ParamField path="REDIS_HOST" type="string" default="localhost">
  Redis host (alternative to `REDIS_URL`).
</ParamField>

<ParamField path="REDIS_PORT" type="number" default="6379">
  Redis port (alternative to `REDIS_URL`).
</ParamField>

<ParamField path="REDIS_PASSWORD" type="string">
  Redis password (alternative to `REDIS_URL`).
</ParamField>

### Queue Concurrency

<ParamField path="CHAT_QUEUE_CONCURRENCY" type="number" default="5">
  Number of concurrent chat jobs per worker.
</ParamField>

<ParamField path="DEEP_RESEARCH_QUEUE_CONCURRENCY" type="number" default="3">
  Number of concurrent deep research jobs per worker.
</ParamField>

<ParamField path="PAPER_GENERATION_CONCURRENCY" type="number" default="1">
  Number of concurrent paper generation jobs per worker.
</ParamField>

<ParamField path="MAX_CONCURRENT_PAPER_JOBS" type="number" default="3">
  Maximum concurrent paper jobs across all users.
</ParamField>

### Rate Limiting

<ParamField path="CHAT_RATE_LIMIT_PER_MINUTE" type="number" default="10">
  Maximum chat requests per user per minute.
</ParamField>

<ParamField path="DEEP_RESEARCH_RATE_LIMIT_PER_5MIN" type="number" default="3">
  Maximum deep research requests per user per 5 minutes.
</ParamField>

### Bull Board Admin Dashboard

<ParamField path="ADMIN_USERNAME" type="string" default="admin">
  Username for Bull Board admin dashboard.
</ParamField>

<ParamField path="ADMIN_PASSWORD" type="string">
  Password for Bull Board admin dashboard.
  
  **Required in production** when `USE_JOB_QUEUE=true`.
</ParamField>

## Character Configuration

See [Character](/configuration/character) for detailed configuration.

<ParamField path="CHARACTER_FILE" type="string" default="characters/bios.json">
  Path to character configuration JSON file.
  
  Available characters:
  - `characters/bios.json` - Default BIOS scientific research assistant
  - `characters/aubrai.json` - Aubrai (Dr Aubrey de Grey persona for longevity research)
</ParamField>

<ParamField path="CHARACTER_JSON" type="string">
  Inline JSON string for character configuration (alternative to `CHARACTER_FILE`).
  
  Useful for simple deployments:
  ```bash
  CHARACTER_JSON='{"name":"MyAgent","system":"You are MyAgent..."}'
  ```
</ParamField>

## Example Configuration

<CodeGroup>

```bash Development
# Basic development setup
NODE_ENV=development
AUTH_MODE=none
PORT=3000

# Database
SUPABASE_URL=your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key

# LLM
OPENAI_API_KEY=sk-...
REPLY_LLM_PROVIDER=openai
REPLY_LLM_MODEL=gpt-4

# In-process execution (no Redis needed)
USE_JOB_QUEUE=false
```

```bash Production
# Production setup
NODE_ENV=production
AUTH_MODE=jwt
BIOAGENTS_SECRET=your-secure-secret-here
ALLOWED_ORIGINS=https://app.yourdomain.com
PORT=3000

# Database
SUPABASE_URL=your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_KEY=your-service-key

# LLM
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
REPLY_LLM_PROVIDER=openai
REPLY_LLM_MODEL=gpt-4

# Job Queue
USE_JOB_QUEUE=true
REDIS_URL=redis://your-redis-host:6379
ADMIN_PASSWORD=your-secure-admin-password

# Storage
STORAGE_PROVIDER=s3
AWS_ACCESS_KEY_ID=your-access-key
AWS_SECRET_ACCESS_KEY=your-secret-key
AWS_REGION=us-east-1
S3_BUCKET=your-bucket-name
```

</CodeGroup>
